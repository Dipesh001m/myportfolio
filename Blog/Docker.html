What is Docker?

Docker is a containerization platform for developing, packaging, shipping and running applications in isolated environments called containers.

What are Docker Images?

Docker images are executable packages bundled with application code, dependencies, and configuration needed to run containers.

What is a Container?

A container packages an application with all necessary dependencies and configurations, enabling reliable deployment across platforms.

Docker Container Lifecycle

  • Created: Container is created but not started.
  • Running: Container is running and processes are active.
  • Paused: Container processes are paused.
  • Stopped: Container is stopped.
  • Deleted: Container is removed from the system.

DockerFile and Image Registry

Dockerfile: A text file that contains instructions to build a Docker image.

Docker Image Registry: A centralized place to store and share Docker images. Examples include Docker Hub and Amazon ECR.

Docker Components

  • Docker Client: CLI to interact with Docker Daemon.
  • Docker Host: Machine running the Docker daemon.
  • Docker Registry: Stores Docker images. Docker Hub is the most popular registry.

Docker Hub vs Docker Dashboard

Docker Hub: Cloud-based image storage and sharing platform.

Docker Dashboard: Local GUI to manage containers, images, and resources.

Docker Login

Use docker login to authenticate with Docker registries using your credentials.

Daemon Logging vs Container Logging

  • Scope:
    • Daemon Logging: Docker Engine/Daemon activities
    • Container Logging: Application/container-specific logs
  • Purpose:
    • Daemon Logging: Tracks system-level Docker operations
    • Container Logging: Tracks application-level output
  • Default Location:
    • Daemon Logging: /var/log/docker.log (or similar)
    • Container Logging: /var/lib/docker/containers/
  • Usage:
    • Daemon Logging: Debugging Docker Engine issues
    • Container Logging: Debugging application behavior

Communication Between Docker Host and Linux Host

  • Bridge Network: Access via Docker host IP and port.
  • Host Network: Direct access using localhost.
  • Port Binding: -p 8080:80 option binds container port to host.
  • SSH: Access Docker host from Linux host using SSH.

What is Docker Compose?

Tool for running multi-container applications using a YAML configuration file.

Example: Running NGINX and Redis together using docker-compose.yml.

Docker Compose Commands

  • docker-compose up – Start all services
  • docker-compose down – Stop all services
  • docker-compose up -d – Detached mode
  • docker ps – List processes
  • docker-compose -f docker-compose.json up – Use JSON format

Docker Basic Commands

  • Check Version: docker –version
  • List Images: docker images
  • Build Image: docker build -t name:tag .
  • Run Container: docker run -d -p 8080:80 image
  • Stop/Start: docker stop/start container_id
  • Logs: docker logs container_id

Docker Push to Hub

docker login
docker tag my-image:latest username/repo:latest
docker push username/repo:latest

Difference Between Docker Image and Layer

Image: Complete, read-only package with all app components.

Layer: Intermediate filesystem changes from Dockerfile instructions.

CMD vs ENTRYPOINT

CMD: Default arguments, overridable.

ENTRYPOINT: Fixed command, not overridable.

Best Practices for Dockerfile

  • Use Alpine or minimal base images.
  • Use multi-stage builds.
  • Combine RUN commands to reduce layers.
  • Use .dockerignore to exclude unnecessary files.

Dockerfile Instructions (Node.js Application)

FROM node:18

# Set working directory
WORKDIR /app

# Copy package.json and package-lock.json
COPY package*.json ./

# Install dependencies
RUN npm install

# Copy application source
COPY . .

# Expose application port
EXPOSE 3000

# Command to run the application
CMD ["node", "app.js"]

LABEL maintainer="dipesh@example.com"

What is Scratch?

An empty, minimal base image often used for statically compiled binaries.

What is Alpine Image?

A lightweight Linux distribution (~5MB) used to build small and secure Docker images.

It is ideal for microservices and containerized applications due to its minimal footprint.

Example for Node.js Application:

FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "app.js"]

What is Distroless Image?

A minimal Docker image without package managers or shells; only app runtime and dependencies. It contains only the essential components required to run your application.

  • Reduces image size significantly.
  • Improves security by minimizing attack surface.
  • No shell or package manager (e.g., no apt, bash).
  • Only includes the runtime and dependencies your app needs.
  • Ideal for production environments.
  • Not suitable for debugging inside containers.

What is Docker Stack?

A Docker Stack is a collection of services that make up an application. It is used to deploy and manage multi-container applications in a Docker Swarm environment, where containers run across multiple hosts in a cluster. it is defined using a Compose file docker-compose.yml.

What is Docker Swarm?

Docker Swarm is Docker’s native clustering and orchestration tool that allows you to manage and deploy a group of Docker containers across multiple hosts as a single virtual system. It is designed for container orchestration, making it easier to scale and manage services in a distributed environment.

Difference Between Docker and Virtual Machines

  • Architecture: Docker shares the host OS kernel, while VMs run a full guest OS on a hypervisor.
  • Startup Time: Docker containers start in seconds; VMs take minutes to boot.
  • Resource Usage: Docker uses fewer resources; VMs consume more due to full OS overhead.
  • Isolation: Docker provides process-level isolation; VMs provide hardware-level isolation.
  • Performance: Docker offers near-native performance; VMs are slower due to virtualization.
  • Portability: Docker containers are highly portable; VMs are less portable and depend on the hypervisor.
  • Use Case: Docker is ideal for microservices and lightweight apps; VMs are suited for monolithic apps and full OS testing.

What is Hypervisor?

A hypervisor is software or firmware that creates and manages virtual machines (VMs) by abstracting and allocating physical hardware resources.
