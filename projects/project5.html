Design and Manage Infrastructure with Terraform and GitHub Actions

Overview

This project automated AWS infrastructure provisioning using Terraform, ensuring consistency and reducing manual errors. It deployed VPCs, EC2 instances, and Application Load Balancer (ALB) across multiple availability zones for high availability and fault tolerance. GitHub Actions was used to integrate CI/CD workflows, enabling seamless deployments and automated infrastructure updates. The outcome was a scalable, reliable, and fully automated cloud environment.

Technologies

Infrastructure as Code (IaC): Terraform

Version Control: GitHub

CI/CD: GitHub Actions

Cloud Provider: AWS (VPC, EC2, ALB, Subnets, Security Groups)

IDE: Visual Studio Code (VSCode)

Deployment Steps

1. Infrastructure Code Development

  • Write Terraform code to define AWS infrastructure (VPC, subnets, internet gateway, security groups, EC2, and ALB).
  • Test Terraform plan locally in VSCode.

2. Push to GitHub Repository

  • Terraform code is pushed to a GitHub repository for version control and CI/CD integration.

3. GitHub Actions CI/CD Pipeline

  • CI/CD pipeline is triggered on every push or pull request.
  • Pipeline runs `terraform init`, `plan`, and `apply` to provision infrastructure automatically.

4. AWS Resource Provisioning

  • Resources are provisioned in AWS in the us-east-1 region across two availability zones.
  • Public subnets, security groups, and internet gateway are configured for external access.
  • EC2 instances are launched and load balanced via ALB.

5. Validation & Scaling

  • Test infrastructure access via ALB DNS name.
  • Terraform allows easy scaling and updates via code modifications and re-deployment.
