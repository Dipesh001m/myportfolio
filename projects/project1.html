AWS 3-Tier Architecture Deployment

Overview

This project sets up a secure and scalable 3-tier web application on AWS using key services: VPC, EC2, RDS, S3, IAM, Route 53, and Certificate Manager. It ensures high availability, security, and fault tolerance.

Technologies

AWS Services: VPC, EC2, RDS, S3, IAM, Route 53, Certificate Manager

Tools: AWS Console, AWS CLI

Security: IAM Roles, SSL Certificates, Private Subnets

Strategy: Multi-AZ deployment

Deployment Steps

1. VPC Setup

Create a custom VPC with public and private subnets for different application tiers.

2. S3 & IAM Role

Upload app code to S3 and create IAM roles with appropriate permissions for EC2 access to the bucket.

3. Database Tier (RDS)

  • Launch an RDS instance (MySQL).
  • Create a database webappdb and a transactions table.
  • Update DbConfig.js with DB credentials and upload it to S3.

4. Application Tier

  • Launch EC2 in a private subnet.
  • Install MySQL client, Node.js, and PM2.
  • Download code from S3, install dependencies, and start the app using PM2.
  • Set up an internal load balancer.
  • Update and upload nginx.conf to route to internal LB DNS.

5. Web Tier

  • Launch EC2 in a public subnet.
  • Install Node.js, and pull frontend build code from S3.
  • Configure Nginx to proxy traffic to internal load balancer.
  • Set up an external load balancer, map domain using Route 53, and apply SSL via ACM.

6. Testing

Open the public IP or domain of the web tier. Test the full app flow by entering data and verifying functionality end-to-end.
