End-to-End DevOps Pipeline for Zomato Application

Overview

Built a complete DevOps pipeline to automate the SDLC for the Zomato app, covering CI, testing, security scanning, containerization, deployment, and monitoring using modern DevOps tools.

Technologies

Version Control: GitHub

CI/CD: Jenkins, ArgoCD

Code Quality: SonarQube

Security Scanning: OWASP Dependency Check, Trivy, Docker Scout

Containerization: Docker, DockerHub

Deployment: Kubernetes (on EC2)

Monitoring: Prometheus, Grafana

Notification: Gmail (Build Status Alerts)

Deployment Steps

1. Code Push

Developer commits code to GitHub, triggering the Jenkins pipeline.

2. CI & Static Analysis

  • Jenkins pulls the code.
  • SonarQube performs code quality analysis.
  • OWASP and Trivy perform vulnerability scans.

3. Docker Build & Scan

  • Docker image is built.
  • Docker Scout scans for vulnerabilities.
  • Image is pushed to DockerHub.

4. CD & Deployment

  • ArgoCD fetches updated image.
  • App is deployed to Kubernetes on Amazon EC2.

5. Monitoring & Notification

  • Prometheus collects metrics.
  • Grafana displays system dashboards.
  • Email notification is sent on build status.
